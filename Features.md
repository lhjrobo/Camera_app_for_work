#features

mainscreen should look and function like an generic camera app
Should be able to choose which camera to use, if there are multiple cameras on the device
Should be able to switch between photo and video mode
Should be able to switch between front and back camera
every time the app is opened, it should create a new folder to store the photos and videos
there should be an UI to select, create, rename and delete the folder to store the photos and videos
If the taken image has qr code inside, it should try to decode it, and ask the user to use the decoded data as the folder name
There is an occasion to take a photo of multiple identical samples. the app should have an option to determine the naming logic of the photos according to the sequence of the photos taken
    for example, if the user takes 10 photos, the photos should be named as 001.jpg, 002.jpg, 003.jpg, 004.jpg, 005.jpg, 006.jpg, 007.jpg, 008.jpg, 009.jpg, 010.jpg
    if the photos should be grouped by the time they are taken, the photos should be named as 001-1.jpg, 001-2.jpg, 001-3.jpg, 002-1.jpg, 002-2.jpg, 002-3.jpg, 003-1.jpg, 003-2.jpg, 003-3.jpg, 004-1.jpg, 004-2.jpg, 004-3.jpg, 005-1.jpg, 005-2.jpg, 005-3.jpg, 006-1.jpg, 006-2.jpg, 006-3.jpg, 007-1.jpg, 007-2.jpg, 007-3.jpg, 008-1.jpg, 008-2.jpg, 008-3.jpg, 009-1.jpg, 009-2.jpg, 009-3.jpg, 010-1.jpg, 010-2.jpg, 010-3.jpg
